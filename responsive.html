响应式设计

data

响应式设计Responsive design，意在实现不同屏幕分辨率的终端上浏览网页的不同展示方式。通过响应式设计能使网站在手机和平板电脑上有更好的浏览阅读体验。之前自己做了一个响应式网站的demo，在这里回顾一下响应式设计中所需要用到的技术：

1、设置 Meta 标签

大多数移动浏览器将html页面放大为宽的视图（viewport）以符合屏幕分辨率。你可以使用视图的meta标签来进行重置。下面的视图标签告诉浏览器，使用设备的宽度作为视图宽度并禁止初始的缩放。在<head>;标签里加入这个meta标签。

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">

2、布局尺寸设置

整体布局时，板块的宽度应使用百分比设置，使其能够适应屏幕的宽度。

.container{
	width: 100%;	
}
.container-content{
	width: 80%;
}

对于字体大小、内外边距等具体长度的设置时应使用rem为单位。使用rem可以避免用户改变浏览器字体后造成的布局混乱。

.footer-content {
	font-size: 1.4rem;
	line-height: 2.4rem;
	margin:0 1rem;
}

3、断点选取

首先根据项目需求确定断点，然后运用媒介查询@media来设置不同分辨率下的样式，如下代码所示：

@media only screen and (max-width: 50em){
	header .top ul li a{
			padding: 0 3rem;
	}
}

@media only screen and (min-width: 30em) and (max-width: 50em){
	header .top ul li a{
			padding: 0 2rem;
	}
}

@media only screen and (max-width: 30em){
	header .top ul li a{
			padding: 0 1rem;
	}
}

需要注意的是@media设置宽度等长度时所使用的单位em，此时的1em的值为浏览器默认的字体大小，一般为16px。

4、处理图片的方法

在不同的分辨率下所需要加载的图片也不同。在小屏幕下需要加载小图片，否则会造成流量的浪费也会增加加载的时间；在大屏幕下需要加载大图片，有利于用户的浏览体验。所以在不同的屏幕下加载合适大小的图片很重要。HTML5为我们提供了<picture>和<source>标签，可以让浏览器自己选择合适的图片加载。代码如下：

<picture>
	<source srcset="img-l.png" media="(min-width:50em)">
	<source srcset="img-m.png" media="(min-width:30em)">
	<img srcset="img.png" alt="img">
</picture>

同样<source>标签中的长度单位em也为浏览器默认的字体大小，一般为16px。
